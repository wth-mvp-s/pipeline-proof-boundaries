

# E002 — Frontière Terraform / Pipeline

Ce document décrit la séparation des responsabilités
entre l’infrastructure et la construction applicative.

1. Terraform définit l’existence, pas le comportement  
   Terraform crée les ressources, les identités et le réseau,
   puis expose des faits techniques (ID, IP, DNS).
   Il ne construit pas l’application et n’intervient pas dans son contenu.

2. Le pipeline construit l’artefact une seule fois  
   Le pipeline produit le paquet applicatif final.
   Une fois le build terminé, la signification des fichiers
   ne doit plus être modifiée.

3. Les outputs sont un contrat, pas de la logique  
   Les données de sortie de Terraform sont consommées par le pipeline
   en lecture seule.
   Elles décrivent l’environnement, sans en piloter la définition.

4. La publication est un simple transport  
   Le transfert de l’artefact (zip, rsync, blob)
   n’introduit aucune nouvelle décision ni configuration.

5. Le runtime sert l’artefact, il ne décide pas  
   La VM, le serveur web ou le conteneur se contentent
   d’exposer l’artefact.
   La logique applicative ne doit pas dépendre de redémarrages
   ni de scripts de démarrage.
