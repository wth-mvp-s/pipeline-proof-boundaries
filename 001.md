

# E001 — Définir l’artefact

Définir l’artefact, c’est définir le périmètre de responsabilité du pipeline.
Dans un processus automatisé impliquant le code, le pipeline et le runtime, cela permet, en cas d’incident, de se situer rapidement et de concentrer l’analyse sur le périmètre réellement contrôlé.
